# Documentação do Projeto Market Hub

Este diretório centraliza toda a documentação técnica, de design e de negócio do projeto, organizada por responsabilidades.

## Estrutura de Diretórios

### 📁 [product/](./product)
Documentação de visão, modelo de negócio e requisitos de alto nível.
- `VISION.md`: Documento de visão do projeto, objetivos e público-alvo.

### 📁 [design/](./design)
Padrões visuais, experiência do usuário e tokens de design.
- `DESIGN_SYSTEM.md`: Guia de estilos, cores, tipografia e componentes.
- `LAWS_OF_UX.md`: Princípios de UX aplicados ao projeto.

### 📁 [architecture/](./architecture)
Decisões estruturais, diagramas e guias arquiteturais.
- `DATABASE_SCHEMA.md`: Estrutura do banco de dados (schema-per-tenant).
- `FRONTEND_MULTI_TENANT.md`: Guia de implementação de subdomínios e white-label.
- `CONSTITUTION.md`: Regras fundamentais e mandatos do projeto.

### 📁 [specs/](./specs)
Especificações técnicas detalhadas para implementação de infraestrutura e ferramentas.
- `DOCKER.md`: Especificação para containerização.
- `MAKEFILE.md`: Especificação para automação de comandos.

*Nota: Documentos operacionais e de sprint continuam na pasta `/context` para facilitar o acesso de agentes de IA.*
